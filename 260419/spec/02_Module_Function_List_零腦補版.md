# 爬蟲寵愛+ Module / Function List（以首頁手繪圖為主）

> 優先規則：首頁 IA、底部導覽、會員位置、AI / 細節入口，以手繪圖為主；其餘欄位與功能細節再沿用 Spec 文件。

---

## 0. 全域交互規則
### Navigation
- 底部固定 5 個 Tab：爬寵 / 討論 / 首頁 / 商店 / 會員
- 底部導覽列需固定可見，不因頁面上下滑而消失
- 首頁內容需可下滑
- 商店 Block 不應該在首頁首屏就全部看完，需往下滑才看到
- 所有子頁面上一頁按鈕固定在左上角
- 所有子頁面支援左滑返回

### Visual / Mock presentation
- Mock 最外層需包一層 iPhone 裝置框
- 畫面內容在手機框內滾動
- 底部導覽列貼齊手機框下方並固定

## 1. 首頁
### Modules
- Today Widget
- 寵 AI+ 助手入口
- 第一排快捷
- 第二排快捷
- 日曆 Block
- 討論 Block
- 商店 Block
- 7 天內待辦區塊

### Functions
- Today Widget 顯示今日任務摘要
- 首屏只先露出 2 筆
- 超過 2 筆時，需在 Today Widget 區塊內直向下滑才能看到其餘卡片
- 卡片欄位：頭像 / 寵物名 / 任務類型 / 時間 / 狀態
- 點任務卡後，預設進該寵物個體頁
- Today Widget 可連到寵 AI+ 助手
- AI 按鈕需為首頁較寬的橫按鈕
- 第一排快捷：我的爬寵 / 餵食／紀錄
- 第二排快捷：餵食計畫 / 繁孕紀錄 / 爬寵相簿
- 首頁內容為長頁面，可向下滑動
- 日曆 Block 顯示整個 4 月月曆
- 月份上方需有圖例：紅點＝餵食、藍點＝其他
- 餵食事件用紅點，其他事件用藍點
- 點首頁月曆可進完整日曆
- 商店 Block 排在更下方，需滑動後才看到
- 7 天內待辦顯示整週內容，不再只顯示單日事件

## 3. 我的爬寵
### Modules
- 寵物列表
- 搜尋 / 篩選
- 新增寵物表單
- 編輯寵物表單
- 愛寵筆記

### Functions
- 新增寵物欄位需包含基本資料與進階資料
- 我的爬寵頁的新增寵物按鈕需為右下角固定按鈕（FAB）
- 顯示頭像、名稱、種類 / 品種標籤、年齡、到家幾天
- 新增寵物的名稱與詳細品種需可手動輸入
- 儲存寵物後需顯示完成提示，返回上一頁，並在前端看到剛儲存的內容
- 愛寵筆記需可進編輯寵物
- 愛寵筆記需作為四大核心功能總入口
- 愛寵筆記需包含摘要、分開的圖表區塊、AI block、歷史、爬寵相簿入口
- 愛寵筆記是寵物資料的主視圖，其他功能回跳時優先回這裡

---

## 4. 餵食／紀錄
### Modules
- 寵物選擇器
- 日期時間區
- 基本欄位區
- 動態快選表單區
- 特殊情況與事件區
- 排泄健康追蹤區
- 自定義常規標籤區
- 照片上傳區
- 儲存後提示 / 警示區

### Functions
- 所有欄位皆為選填
- 先選寵物，再動態切換物種表單
- 不顯示依寵物直接帶入物種模板器，應直接依所選寵物帶入對應物種模板
- 表單以乾淨的下拉式選單為主，不要過度雜亂
- 日期時間預設顯示當下時間，但使用者可調整幾點
- 支援照片與拍照
- 支援一次記錄多種事件
- 餵食內容 / 事件 / 排泄的「其他」需可手動輸入
- 只有在用戶輸入其他文字後，才顯示「儲存成自定義標籤」按鈕
- 輸入後可儲存為自定義標籤沿用
- 餵食方式 / 事件 / 排泄需收在進階選項內，以 toggle 展開
- 高危險事件勾選後立即顯示警示
- 頁面底部需有送出按鈕
- 點送出後需顯示完成提示，返回剛剛來源頁；若由愛寵筆記進入，則回愛寵筆記
- 只儲存用戶本次有輸入 / 有選擇的欄位，未填欄位不寫入、不顯示
- 送出後需同步更新：最新事件、歷史紀錄、時間軸、相關待辦狀態

### 動態快選表單種類
- 蛇類
- 守宮／蜥蜴
- 陸龜
- 水龜
- 兩棲
- 節肢

## 5. 日曆
### Modules
- 月曆
- 日檢視
- 寵物篩選器
- 事件類型篩選器
- 當日事件列表
- 事件詳情卡

### Functions
- 顯示所有照護事件總視圖
- 日曆主頁預設顯示整個月的月曆
- 月份上方需有圖例：紅點＝餵食、藍點＝其他
- 月曆點點規則：餵食紅點，其他事件藍點
- 不提供月曆 / 日檢視切換按鈕，固定以月曆總覽呈現
- 支援依寵物與依事件類型過濾
- 點某日可看當日所有事件與任務
- 點某事件可看詳情
- 可跳到該筆紀錄或該隻寵物頁
- 下方顯示近期待辦內容

## 5.5 AI 助手
### Modules
- 首頁寵 AI+ 助手按鈕
- 個體頁 AI 助手 Block
- AI 對話窗
- 頂部爬寵切換器
- 對話內容區
- 圖片上傳區
- 輸入區

### Functions
- 首頁 AI 按鈕名稱固定為 `寵 AI+ 助手`
- 個體頁需有 AI 內容摘要區塊，放在體重變化圖上方
- 個體頁右下角需有寵AI+ FAB
- AI 區塊只顯示 AI 的一句話
- 點寵AI+ FAB 可進 AI 對話窗
- AI 對話窗最上方可切換爬寵
- AI 對話窗需可上傳照片
- AI 對話窗返回需回上一個進入脈絡
- 對話需根據該爬寵資料進行
- AI 區塊與 AI 對話窗都需反映該寵物最新狀態 / 待辦脈絡

## 6. 討論
### Modules
- 討論列表頁
- 發文編輯器
- 文章內頁
- 回覆區

### Functions
- 列表顯示：預覽圖 / 標題 / 摘要 / 日期 / 作者 / 愛心數 / 回覆數
- 文章左側需有預覽圖
- 發文按鈕固定在右下角（FAB）
- 支援發文、草稿儲存、分類選擇、文中插圖、封面圖
- 發文編輯器不保留格式工具列
- 文中圖片上傳需顯示數量上限，例如 (0/5)
- 發文圖片上傳 mock 需可預覽、加入、移除
- 封面圖需可切換 mock 素材
- 支援文字回覆與圖片上傳
- 發布後文章內頁需呈現更完整的文章排版感，不只是單一文字堆疊
- 文章需有留言按鈕
- 文章需可按愛心，且同一人對同一篇文只能按一次
- 自己發的文章前端需可點擊編輯
- 留言區樣式與留言 mock data 需存在，送出後需即時顯示
- 列表頁與文章內頁都需顯示正確的愛心數 / 留言數
- 文章圖片顯示結構為封面圖 + 文中圖片橫向瀏覽
- 發布功能需真的有作用

## 7. 商店
### Modules
- 商店首頁
- 商品列表頁
- 商品詳情頁
- 購物車
- 結帳頁
- 訂單完成頁

### Functions
- 第一期僅平台自營商品
- 商品卡需顯示商品圖，不可只有文字或 Banner
- 商店頁右下角固定購物車入口（FAB / fixed CTA）
- 支援加入購物車
- 加入購物車後需實際新增商品數量
- 購物車可跳轉結帳頁
- 結帳頁需有地址 / 聯絡資訊欄位
- 結帳欄位需可實際輸入
- 支援一次結帳
- 商品欄位：商品名稱 / 商品分類 / 封面圖 / 輪播圖 / 價格 / 描述 / 規格 / 庫存 / 出貨資訊

## 8. 會員
### Modules
- 我的資料
- 我的爬寵摘要
- 我的文章
- 我的訂單
- 通知設定（單一開關）
- 登入 / 註冊

### Functions
- 通知設定歸在會員內
- 會員需有登入 mock
- 登入方式：Email 驗證 / Google / Apple OAuth
- 登入後需可切換我的資料 / 我的爬寵摘要 / 我的文章 / 我的訂單 / 通知設定
- 通知設定只保留一個 `開啟通知` 開關，點擊後導向 APPLE 設定頁面
- 會員各分頁需有基本互動，不可只有靜態標題

## 8.5 雲端資料儲存 / 後端方向
### Modules
- 會員資料儲存
- 寵物資料儲存
- 照護紀錄 / 餵食計畫儲存
- 討論資料儲存
- 購物車 / 訂單 / 收件資訊儲存

### Functions
- 用戶資料需存在雲端，不可只存在前端本地
- 雲端資料需覆蓋會員、寵物、紀錄、討論、購物資訊
- mock 階段可先前端模擬，但架構上需預留未來落雲端
- 後續需專門選型後端 / 雲服務方案
- 可先列入評估：Cloudflare + AWS
