# 爬蟲寵愛+ Sitemap（以首頁手繪圖為主）

> 優先規則：首頁 IA、底部導覽、會員位置、AI / 細節入口，以手繪圖為主；其餘欄位與功能細節再沿用 Spec 文件。

---

## 1. 全域導覽與交互規則

### 1.1 底部導覽列（固定可見）
固定 5 個 Tab：
1. 爬寵
2. 討論
3. 首頁
4. 商店
5. 會員

### 1.2 全域交互
- 首頁為可下滑長頁
- 內容會在底部導覽列上方持續往下延伸
- 底部導覽列從頭到尾固定可見
- 所有子頁面的「上一頁」都在左上角
- 所有子頁面支援左滑返回

### 1.3 會員 / 通知設定
- 會員為底部主 Tab
- 通知設定歸屬於會員入口內

---

## 2. 首頁

### 2.1 上方區塊：Today Widget
- 今日待辦摘要
- 顯示今日任務卡片
- 首屏只先露出 2 筆
- 若超過 2 筆，需在 Today Widget 區塊內直向下滑才能看到其餘卡片
- 卡片欄位：頭像 / 寵物名 / 任務類型 / 時間 / 狀態
- 點卡片後，預設進該寵物個體頁；若之後有更明確任務頁，才跳該任務對應頁

### 2.2 Today Widget 延伸入口
- 寵 AI+ 助手

### 2.3 第一排快捷
- 我的爬寵
- 餵食／紀錄

### 2.4 第二排快捷
- 餵食計畫
- 繁孕紀錄
- 爬寵相簿

### 2.5 下方主區塊（往下滑）
- 日曆 Block（4 月整月月曆，餵食事件紅點，其他事件藍點）
- 討論 Block（可點進討論頁）
- 商店 Block（需往下滑才看到）

---

## 3. 爬寵 Tab

### 3.1 爬寵 Hub
- 我的爬寵
- 餵食／紀錄
- 餵食計畫
- 照片與歷史紀錄入口
- 第二期入口預留
  - 爬寵筆記
  - 繁孕計畫
  - 彩虹橋
  - 爬蟲百科

### 3.2 我的爬寵
- 寵物列表
- 新增寵物
- 編輯寵物
- 愛寵筆記
- 愛寵筆記內可進編輯寵物
- 新增 / 編輯儲存後，需立即反映到前端顯示

### 3.3 愛寵筆記
- 區塊 A：個體摘要
- 區塊 B：核心功能入口（餵食／紀錄 / 餵食計畫 / 寵AI+ / 爬寵相簿）
- 區塊 C：AI 摘要一句話
- 區塊 D：體重變化圖
- 區塊 E：餵食頻率圖
- 區塊 F：溫濕度趨勢
- 區塊 G：近期事件時間軸
- 區塊 H：歷史紀錄
- 區塊 I：爬寵相簿入口
- 右下角固定 `寵AI+` FAB
- 愛寵筆記是核心功能回跳優先頁

### 3.4 餵食／紀錄
- 寵物選擇
- 依寵物直接帶入物種模板
- 日期時間
- 基本欄位
- 動態快選表單
- 特殊情況與事件
- 排泄健康追蹤
- 自定義常規標籤
- 照片上傳 / 拍照
- 儲存後提示 / 警示
- 送出後需回到該寵物個體頁，並看到新紀錄已寫入

### 3.5 餵食計畫
- 計畫列表
- 新增計畫
- 編輯計畫
- 日曆切換
- 計畫詳情
- 通知權限引導
- 可由計畫詳情跳到寵物頁
- 計畫屬於長期安排，不使用「完成計畫」語意
- 新增 / 編輯計畫後，需反映到寵物頁與待辦資料

---

### 3.6 AI 助手對話窗
- 頂部爬寵切換器
- 對話內容區
- 圖片上傳按鈕
- 輸入區
- 需顯示目前該寵物的最新狀態摘要
- 根據所選爬寵資料進行對話
- 返回需回上一個進入脈絡，而不是固定回首頁

- 文章內頁
- 回覆區

- 7 天內待辦區塊
