# 爬蟲寵愛+ Module / Function List（以首頁手繪圖為主）

> 優先規則：首頁 IA、底部導覽、會員位置、AI / 細節入口，以手繪圖為主；其餘欄位與功能細節再沿用 Spec 文件。

## 0. 全域交互規則
### Navigation
- 底部固定 5 個 Tab：爬寵 / 討論 / 首頁 / 商店 / 會員
- 底部導覽列需固定可見，不因頁面上下滑而消失
- 首頁內容需可下滑
- 商店 Block 不應該在首頁首屏就全部看完，需往下滑才看到
- 所有子頁面上一頁按鈕固定在左上角
- 所有子頁面支援左滑返回

### Visual / Mock presentation
- Mock 最外層需包一層 iPhone 裝置框
- 畫面內容在手機框內滾動
- 底部導覽列貼齊手機框下方並固定

- 點首頁月曆可進完整日曆
- 商店 Block 排在更下方，需滑動後才看到
- 7 天內待辦顯示整週內容，不再只顯示單日事件

## 3. 我的爬寵
### Modules
- 寵物列表
- 搜尋 / 篩選
- 新增寵物表單
- 編輯寵物表單
- 愛寵筆記

### Functions
- 新增寵物欄位需包含基本資料與進階資料
- 我的爬寵頁的新增寵物按鈕需為右下角固定按鈕（FAB）
- 顯示頭像、名稱、種類 / 品種標籤、年齡、到家幾天
- 新增寵物的名稱與詳細品種需可手動輸入
- 儲存寵物後需顯示完成提示，返回上一頁，並在前端看到剛儲存的內容
- 愛寵筆記需可進編輯寵物
- 愛寵筆記需作為四大核心功能總入口
- 愛寵筆記需包含摘要、分開的圖表區塊、AI block、歷史、爬寵相簿入口
- 從愛寵筆記進入爬寵相簿時，需進入同一個爬寵相簿頁，但右上 filter 預設為該寵物
- 愛寵筆記是寵物資料的主視圖，其他功能回跳時優先回這裡

## 4. 餵食／紀錄
### Modules
- 寵物選擇器
- 日期時間區
- 基本欄位區
- 動態快選表單區
- 特殊情況與事件區
- 排泄健康追蹤區
- 自定義常規標籤區
- 照片上傳區
- 儲存後提示 / 警示區

### Functions
- 所有欄位皆為選填
- 先選寵物，再動態切換物種表單
- 不顯示依寵物直接帶入物種模板器，應直接依所選寵物帶入對應物種模板
- 表單以乾淨的下拉式選單為主，不要過度雜亂
- 日期時間預設顯示當下時間，但使用者可調整幾點
- 支援照片與拍照
- 在愛寵筆記新增的照片，需自動標記為該寵物照片
- 在餵食／紀錄新增的照片，需依所選寵物自動標記照片歸屬
- 支援一次記錄多種事件
- 餵食內容 / 事件 / 排泄的「其他」需可手動輸入
- 只有在用戶輸入其他文字後，才顯示「儲存成自定義標籤」按鈕
- 輸入後可儲存為自定義標籤沿用
- 餵食方式 / 事件 / 排泄需收在進階選項內，以 toggle 展開
- 高危險事件勾選後立即顯示警示
- 頁面底部需有送出按鈕
- 點送出後需顯示完成提示，返回剛剛來源頁；若由愛寵筆記進入，則回愛寵筆記
- 只儲存用戶本次有輸入 / 有選擇的欄位，未填欄位不寫入、不顯示
- 送出後需同步更新：最新事件、歷史紀錄、時間軸、相關待辦狀態

### 動態快選表單種類
- 蛇類
- 守宮／蜥蜴
- 陸龜
- 水龜
- 兩棲
- 節肢

## 5.5 AI 助手
### Modules
- 首頁寵 AI+ 助手按鈕
- 個體頁 AI 助手 Block
- AI 對話窗
- 頂部爬寵切換器
- 對話內容區
- 圖片上傳區
- 輸入區

### Functions
- 首頁 AI 按鈕名稱固定為 `寵 AI+ 助手`
- 個體頁需有 AI 內容摘要區塊，放在體重變化圖上方
- 個體頁右下角需有寵AI+ FAB
- AI 區塊只顯示 AI 的一句話
- 點寵AI+ FAB 可進 AI 對話窗
- AI 對話窗最上方可切換爬寵
- AI 對話窗需可上傳照片
- AI 對話窗返回需回上一個進入脈絡
- 對話需根據該爬寵資料進行
- AI 區塊與 AI 對話窗都需反映該寵物最新狀態 / 待辦脈絡

## 6. 討論
### Modules
- 討論列表頁
- 發文編輯器
- 文章內頁
- 回覆區

### Functions
- 列表顯示：預覽圖 / 標題 / 摘要 / 日期 / 作者 / 愛心數 / 回覆數
- 列表頁愛心數 / 回覆數需直接反映文章目前資料，不可與文章內頁脫鉤
- 文章左側需有預覽圖
- 發文按鈕固定在右下角（FAB）
- 支援發文、草稿儲存、分類選擇、文中插圖、封面圖
- 發文編輯器不保留格式工具列
- 文中圖片上傳需顯示數量上限，例如 (0/5)
- 發文圖片上傳 mock 需可預覽、加入、移除
- 封面圖需可切換 mock 素材
- 支援文字回覆與圖片上傳
- 發布後文章內頁需呈現更完整的文章排版感，不只是單一文字堆疊
- 文章頂部需顯示作者 / 日期 / 分類 / 愛心數 / 留言數 / 編輯入口（若為本人）
- 文章需有留言按鈕
- 文章需可按愛心，且同一人對同一篇文只能按一次
- 自己發的文章前端需可點擊編輯
- 留言區樣式與留言 mock data 需存在，送出後需即時顯示
- 列表頁與文章內頁都需顯示正確的愛心數 / 留言數
- 愛心與留言在列表與內頁皆使用 icon + 數字顯示
- 文章圖片顯示結構為封面圖 + 文中圖片橫向瀏覽
- 發布功能需真的有作用

## 13. 前端實作補齊（2026-04-26）
> 依 `src/App.tsx` 目前可操作 mock 前端補齊。以下項目為前端已出現、但原 spec 未明確寫完整或只散落提及的功能與互動邏輯。

### 13.1 全域導航與返回
- App 採單一 `screen` 狀態切換頁面，主要 Tab 為：爬寵、討論、首頁、商店、會員。
- 底部 Tab 切換會清空返回堆疊；子頁返回使用 `backStack` 回到前一頁，若沒有上一頁則回首頁或指定 fallback。
- 多數子頁支援左側滑動返回：起點需在畫面左側 80px 內、水平位移大於 120px、垂直位移小於 100px 時觸發返回。
- 底部 Tab 高亮會依子頁歸屬映射：寵物子頁歸爬寵、文章/編輯器歸討論、商品/購物車/結帳歸商店、日曆/AI 歸首頁、登入歸會員。
- 浮動操作按鈕依頁面切換：討論頁顯示「發文」、商店顯示「購物車（數量）」、我的爬寵顯示「新增寵物」、寵物個體頁顯示「寵AI+」。

### 13.2 首頁補齊
- 首頁可直接進入：我的爬寵、餵食／紀錄、餵食計畫、討論區、商店、爬寵相簿、日曆、AI 助手、施工中頁。
- 今日任務卡點擊後會開啟對應寵物個體頁。
- 首頁文章卡會開啟文章詳情；商品卡會開啟商品詳情。
- 首頁內的月曆點選日期會更新全域 `selectedDate`，日曆頁沿用同一日期狀態。

### 13.3 爬寵與個體頁補齊
- 我的爬寵列表點選卡片會進入個體頁，並設定目前選取寵物。
- 新增寵物會新增到寵物陣列最前方、刷新 AI 摘要、選取該寵物、提示儲存成功，並返回列表。
- 編輯寵物會覆蓋原寵物資料、刷新 AI 摘要、提示儲存成功，並返回個體頁。
- 個體頁提供：編輯資料、餵食／紀錄、餵食計畫、寵AI+、爬寵相簿、查看更多歷史紀錄。
- 個體頁包含體重折線、餵食頻率圖、濕度折線、近期時間軸、照片區。

### 13.4 相簿補齊
- 爬寵相簿為獨立頁面，可從首頁、爬寵 Hub、個體頁進入。
- 入口若從個體頁進入，預設只看該寵物；若從首頁或 Hub 進入，預設全部寵物。
- 相簿支援寵物篩選：全部寵物或單一寵物。
- 點照片會開啟放大檢視，顯示照片與目前歸屬寵物。
- 放大檢視中可改照片歸屬寵物；前端會把照片從所有寵物移除，再加入新寵物相簿最前方。
- 相簿頁有「上傳」浮動按鈕；若目前篩選為全部，預設上傳到第一隻寵物。

### 13.5 歷史紀錄與施工中頁補齊
- 歷史紀錄為個體頁延伸子頁，列出該寵物所有 records，並標示異常與有照片的紀錄。
- 施工中頁為首頁可觸發的 mock 子頁，僅顯示尚未開放狀態與返回。

### 13.6 餵食／紀錄補齊
- 紀錄頁可選寵物、選小時、選餵食內容，並輸入餵食重量與數量。
- 紀錄 preset 依物種大類切換，包含蛇、守宮、龜、澤龜、兩棲、節肢等分類。
- 「進階紀錄」可展開或收起，內容包含事件與排泄狀態。
- 餵食內容、事件、排泄皆支援「其他」；輸入文字後可儲存成自定義標籤並立即選用。
- 上傳照片與拍照在紀錄表單內為 mock alert；送出時若有照片相關紀錄，會把 mock 圖片加入該寵物相簿。
- 送出紀錄會新增到寵物 records 最前方、同步新增 timeline、更新最新事件；餵食紀錄會更新最近餵食日期。
- 送出紀錄後會嘗試把該寵物未完成的餵食／泡水／觀察計畫標記為完成，刷新 AI 摘要並返回個體頁。

### 13.7 餵食計畫與日曆補齊
- 計畫頁有 list、settings、create、edit 四種模式。
- 計畫可新增與編輯；可選寵物、任務類型、頻率、時間。
- 任務類型支援「其他」，可儲存成自定義計畫類型。
- 頻率支援「每間隔數天」與「每週特定時間」；每週模式可多選星期。
- 新增計畫會依排程產生一或多筆 calendar event，更新寵物 AI 摘要並提示成功。
- 編輯計畫會更新原 event 的寵物、類型、標題、頻率、日期、時間。
- 計畫清單可選取單一計畫查看詳情；詳情可前往寵物頁或開啟通知權限引導。
- 日曆頁支援寵物篩選與事件類型篩選；選日期後列出該日事件，點事件後顯示詳情。
- 日曆事件詳情可前往寵物頁或前往紀錄頁。
- 月曆用紅點表示餵食事件，用藍點表示非餵食事件。

### 13.8 AI 助手補齊
- AI 助手頁會保留來源返回目標；從首頁或個體頁進入時，返回目標不同。
- AI 助手可切換寵物，並顯示該寵物最新事件與 AI 摘要。
- 對話內容來自 mock conversation；輸入框與送出按鈕目前僅呈現 UI，不會新增訊息。

### 13.9 討論區與文章補齊
- 討論列表點文章進入文章詳情。
- 討論頁浮動按鈕進入發文編輯器。
- 編輯器包含標題、封面圖、附圖、正文；封面圖按鈕會在 mock 圖片間輪替。
- 附圖上傳最多保留 5 張；點已加入附圖可移除。
- 草稿儲存目前為 mock alert。
- 發布時標題與正文必填；缺漏會 alert。
- 新發文會建立 mock post、加入討論列表最前方、初始化留言陣列、清空草稿並進入文章詳情。
- 若從自己的文章按「編輯」，會帶入原文章資料；發布後更新原文章並回文章詳情。
- 文章愛心每篇每次 session 只能按一次；重複按會 alert。
- 留言送出前會 trim，空白留言會 alert；成功後新增在留言最前方、更新回覆數並清空輸入框。

### 13.10 商店、購物車與結帳補齊
- 商店列表可開商品詳情，也可直接加入購物車。
- 加入購物車若商品已存在則數量加一；不存在則新增一筆 cart line。
- 商品詳情包含規格、庫存、運送說明；可加入購物車或立即購買。
- 立即購買會先加入購物車再進入結帳頁。
- 購物車頁列出商品、數量、小計與總計；目前不提供刪除與調整數量。
- 結帳頁包含收件人、手機、縣市／地區、地址、備註與訂單總計。
- 送出訂單目前為 mock alert，不會清空購物車。

### 13.11 會員與登入補齊
- 未登入會員頁顯示登入引導；按鈕進入登入／註冊頁。
- 登入頁包含 Email、密碼、暱稱、Google、Apple；按登入會把 `isLoggedIn` 設為 true 並返回會員頁。
- 已登入會員頁有分頁：會員資料、我的爬寵、我的文章、我的訂單、通知設定。
- 會員資料可編輯名稱與 Email；儲存為 mock alert。
- 我的訂單可切換 mock 訂單 `RL+1024` 與 `RL+1021` 顯示不同狀態。
- 通知設定可切換開啟／未開啟 badge，並 mock 跳轉 Apple 設定。

### 13.12 Mock 資料與狀態限制
- 所有資料變更皆為前端記憶體 mock 狀態，重新整理後不保留。
- 圖片來源包含 `/mock-images/`；若不是圖片路徑則以文字 fallback 顯示。
- alert 文字代表 mock 成功、權限或錯誤提示，尚未串接真實 API、登入、付款、通知、相機或相簿權限。
