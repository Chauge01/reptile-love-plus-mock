# 爬蟲寵愛+ 整體 IA 架構圖（設計檢查版）

> 建議直接開 `ia-diagram.html`。HTML 版支援縮放、拖曳、模組聚焦，以及點擊節點或線時固定高亮關聯線與相連節點。

## 設計師使用方式

1. 先用左側「聚焦模組」看單一模組，不要一開始看全圖。
2. 點擊節點或線時，看橘色節點與橘色線，確認這個 screen 的上一層與下一層。
3. 高亮會持續保留，直到點擊其他節點、線、左側模組、空白畫布或「全部顯示」。
4. 看跨模組線時，優先檢查是否為首頁入口、個體頁入口、日曆跳轉、AI 返回脈絡。
5. 若需要標註 UI 工作範圍，可用模組分組：
   - 首頁 Home
   - 爬寵核心 Pet Core
   - 支援工具 Calendar / Photo / AI
   - 討論 Community
   - 商店 Commerce
   - 會員 Member

## 本版整理策略

- 主圖只畫 screen state 與關鍵頁內互動，避免每個小元件都變成節點。
- 同一 screen 內的表單欄位與狀態放在節點註記，不拉獨立線。
- 對設計有影響的頁內互動才保留節點，例如照片放大、留言互動、會員分頁、結帳 mock 送出。
- 已把容易誤解的舊節點改名：
  - `愛寵筆記` -> `寵物個體頁`
  - `訂單完成頁` -> 移除，改註記結帳 mock alert 不切頁
  - `爬寵相簿：全部 / 指定` -> 合併為單一相簿，入口決定 filter

## IA 檢查結果

| 範圍 | 結論 |
|---|---|
| 底部導覽 | 符合 spec 與前端：爬寵 / 討論 / 首頁 / 商店 / 會員。 |
| 首頁 | 補齊快捷、日曆、AI、內容預覽、施工中入口。 |
| 爬寵核心 | 符合前端 screen：Hub、列表、新增、個體、編輯、紀錄、計畫、相簿、歷史、AI。 |
| 日曆 | 符合：整月月曆、篩選、當日事件、事件詳情、跳寵物頁或紀錄頁。 |
| 相簿 | 符合：單一相簿頁，支援全部/指定寵物 filter、照片放大與重新標記。 |
| AI | 符合：可從首頁或個體頁進入，返回依來源脈絡。 |
| 討論 | 符合：列表、文章、編輯器、留言；自己的文章可進編輯器。 |
| 商店 | 修正：前端無訂單完成頁，結帳送出只 alert 並停留。 |
| 會員 | 符合：未登入引導、登入頁、已登入分頁與通知開關。 |
